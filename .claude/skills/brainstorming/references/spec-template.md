# Spec Template

使用下面的骨架写设计文档；可以按任务复杂度增删细节，但不要删掉核心边界。

## 文档标题

`# [主题] 设计文档`

## 背景与目标

- 为什么要做
- 这次要解决什么问题
- 成功标准是什么

## 范围

- 本次明确要做的内容

## 非范围

- 本次不做的内容
- 明确排除哪些扩展项

## 方案比较

### 方案 A

- 核心思路
- 优点
- 缺点

### 方案 B

- 核心思路
- 优点
- 缺点

## 推荐方案

- 为什么选它
- 主要 trade-offs

## 关键边界与组件职责

- 组件或模块 1
- 组件或模块 2

## 数据流 / 控制流

- 关键输入
- 关键处理步骤
- 关键输出

## 错误处理与回退

- 主要失败模式
- 对应处理策略

## 测试策略

- 需要验证的核心行为
- 适合的测试层级

## 未决事项

- 仍需确认的问题

默认路径优先级：用户指定路径 > 仓库已有约定 > `docs/specs/<YYYY-MM-DD>-<topic>-design.md`。
